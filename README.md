# 🚀 Algorithm Study & Engineering Practice

> **"지속 가능한 성장과 협업 역량 향상을 위한 코드 기록"** > 단순히 정답을 맞히는 것을 넘어, 실무 현장에서 즉시 협업 가능한 엔지니어로 성장하기 위한 모든 과정을 기록합니다.

## 📌 Project Objectives

### 1. 코딩 테스트 대비 및 알고리즘 역량 강화
- 다양한 문제 해결을 통해 효율적인 자료구조 및 알고리즘 설계 능력 배양.
- 시간/공간 복잡도 최적화를 고려한 문제 풀이 지향.

### 2. 코드 리뷰 기반의 비판적 사고 훈련
- **강점 포인트**: 실무에서 코드 리뷰는 팀 전체의 기술 부채를 줄이는 핵심 과정입니다. 혼자 학습할 때 발생할 수 있는 **'확증 편향'을 방지**하고, 운영 단계에서 발생할 수 있는 **에지 케이스(Edge Case)를 사전에 차단**하는 습관을 기르기 위해 AI 셀프 리뷰를 수행합니다. 이는 곧 **동료의 리뷰 리소스를 아끼고 고품질의 코드를 생산**할 수 있는 저만의 강력한 무기입니다.

### 2. AI(Gemini) 활용을 통한 코드 리뷰 및 비판적 사고 훈련
- **현업의 코드 리뷰 문화 지향**: 현업에서 코드 리뷰는 단순한 검수를 넘어 기술 부채를 줄이고 팀의 성장을 도모하는 필수적인 과정입니다. 소속된 조직이 없는 취업 준비 환경에서도 이러한 리뷰 문화를 체화하기 위해 AI(Gemini)를 **가상의 시니어 개발자**로 설정하여 협업을 진행합니다.

  - **자기 객관화**: 혼자 학습할 때 빠지기 쉬운 '확증 편향'과 '내 코드에 대한 관성'을 AI의 객관적인 피드백을 통해 타파합니다.
  - **잠재적 리스크 탐색**: 운영 환경에서 치명적일 수 있는 에지 케이스(Edge Case), 오버플로우, 메모리 효율성 등을 사전에 검토하는 습관을 기릅니다.
  - **리뷰받기 좋은 코드**: AI와 소통하며 피드백을 수용하고 코드를 수정하는 과정을 통해, 실무 투입 시 **동료의 리뷰 리소스를 아끼고 고품질의 코드를 생산**할 수 있는 협업 준비성을 증명합니다.

### 3. 실무 지향적 개발 프로세스 체득
- **Naming Convention**: 자바 표준 명명 규칙(CamelCase) 준수 및 매직 넘버 지양.
- **Git Flow**: `main` 브랜치 보호를 위한 `develop/feature` 브랜치 전략 운용.
- **Documentation**: Conventional Commits 및 PR 네이밍 규칙을 적용하여 누구나 이해할 수 있는 히스토리 생성.

### 4. 지속 가능성 증명 (1일 1코딩)
- 취업 준비 기간 중 발생할 수 있는 공백기를 **학습의 연속성**으로 채우며, 매일 기록된 데이터로 꾸준히 성장하는 성실함을 증명합니다.
